# Support

Use the right channel for the right kind of request.

## Use an issue for

- bug reports
- feature requests
- docs improvements
- compatibility requests for new agent runtimes

## Use a private channel for

- security reports
- sensitive compliance or customer material
- anything that should not be public

## Before opening an issue

- include your Node version
- include the exact command you ran
- include the generated error text
- mention whether you used `onboard`, `capture`, `interview`, or `create`

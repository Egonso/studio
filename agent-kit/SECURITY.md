# Security Policy

## Supported versions

Security fixes are handled on the latest released version of the kit.

## Reporting a vulnerability

Please do not open a public issue for a security-sensitive problem.

Instead:

1. Share a private report with the maintainers.
2. Include reproduction details, affected files, and impact.
3. If possible, suggest mitigations or safe defaults.

## What counts as security-sensitive here

- command execution risks
- unsafe path handling or file writes
- accidental disclosure of local config or sensitive prompts
- packaging or marketplace metadata that could mislead users about security guarantees

## Response goals

- Acknowledge the report quickly.
- Reproduce and assess impact.
- Ship a fix or mitigation note.
- Publish a coordinated disclosure summary when appropriate.

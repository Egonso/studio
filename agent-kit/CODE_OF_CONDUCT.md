# Code of Conduct

We want this project to be useful, respectful, and safe for contributors from different backgrounds and disciplines.

## Expected behavior

- Be respectful and specific in feedback.
- Assume good intent, but be honest about risks and gaps.
- Prioritize clarity over ego.
- Keep compliance claims grounded in sources and practical limits.
- Focus discussion on improving the work, not judging the person.

## Unacceptable behavior

- Harassment, intimidation, or personal attacks
- Deliberate misrepresentation of security or compliance claims
- Dismissing contributors for basic questions or different experience levels
- Sharing confidential or sensitive material without permission

## Enforcement

Project maintainers may remove, edit, or reject contributions, comments, or issues that violate this code of conduct.

## Reporting

If something needs attention, open a private security or support contact path instead of escalating publicly. See `SECURITY.md` and `SUPPORT.md`.

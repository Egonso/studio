# Certification Document Template

Date: 2026-05-07

## Scope

Future person certificates for the EU AI Act Officer certification use the
legacy Documentero certificate template instead of the native monochrome PDF
fallback.

Existing certificates are not backfilled or regenerated by this change.
If a user already has an active certificate and repeats the exam, the existing
certificate document is kept.

## Data Flow

1. The user completes all non-download Academy videos.
2. The user passes the integrated certification exam.
3. The server creates or reuses the active `person_certificates` record.
4. For newly issued certificates, the server renders the document through
   Documentero when these runtime variables are configured:
   - `CERTIFICATION_DOCUMENT_PROVIDER=documentero`
   - `CERTIFICATION_ENABLE_DOCUMENTERO=1`
   - `DOCUMENTERO_TEMPLATE_ID=z1ffZUyuWFQwXBaoo2NL`
   - `DOCUMENTERO_API_KEY` as an App Hosting secret with access granted to the
     `studio` backend in `us-central1`
5. The generated PDF URL or archived PDF data URI is stored in
   `certificate_documents` and exposed through the existing public verification
   route.

## Template Fields

The render payload keeps compatibility with the legacy template fields:

- `Vorname_Nachname`
- `Zert_Nr`
- `qrcode`
- `datum`

The current implementation also sends the richer KI-Register fields such as
`verifyUrl`, `validUntil`, `modules`, `company`, and `badgeAssetUrl` for future
template revisions.

## Risks

- Documentero availability affects only new PDF rendering. Certificate records
  and public verification remain in KI-Register.
- If Documentero is not configured, the system falls back to the native PDF
  renderer.
- Signatures are expected to live in the Documentero template, not in the app
  repository.
